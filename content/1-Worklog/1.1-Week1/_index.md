### Week 1 Achievements:

* Understood the basic roles of IAM users, IAM roles, and permission management in AWS.
* Successfully practiced providing Internet connectivity to EC2 through a VPC, public subnet, Internet Gateway, route table, and Elastic IP.
* Clarified the differences among the main networking components:
  * **Internet Gateway:** Enables two-way Internet communication for resources in a public subnet when routing and public addressing are configured correctly.
  * **NAT Gateway:** Enables resources in a private subnet to initiate outbound Internet access without allowing unsolicited inbound connections.
  * **Route table:** Defines the paths used by network traffic in a subnet.
* Accessed EC2 through both standard remote connection methods and AWS Systems Manager Session Manager.
* Used CloudWatch to observe EC2 status and operational metrics.
* Configured budget alerts to control lab expenses.
* Tested Site-to-Site VPN connectivity and identified a one-way routing issue for further investigation.
* Documented an access issue with a Windows EC2 AMI for later troubleshooting.
* Created an initial learning list for the project, including S3, RDS, DynamoDB, Lambda, CloudFront, ECS, SQS, SNS, Cognito, CDK, WAF, KMS, Macie, and Amplify.
